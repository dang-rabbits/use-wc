# Styling

Authoring CSS selectors can be complicated due to the inherint limitations of `part` and other Shadow DOM selectors. To overcome this there are plans to provide an opinionated theming layer built using CSS custom properties (aka, CSS variables). The custom property names will borrow concepts from semantic design token naming conventions. This approach will allow adopting teams to utilize their own design system core and alias tokens.

  --usewc-color-input-background-invalid: #fff;

#### Proposed Naming Conventions

##### Token Categories

- `color`
- `size`
- `space`
- `font`
- `shadow`
- `effect`

##### Component Categories

- `input` - anything that can hold a value
- `selector` - an input that toggles between two or three states (on / off, true / false, checked / unchecked / mixed)
  - `checkbox` - a selector that toggles between checked and unchecked
  - `radio` - a selector that toggles between checked and unchecked, but only one can be checked at a time
  - `switch` - a selector that toggles between checked and unchecked
- `action` - a button or link that triggers an action
  - `button` - a button that triggers an action
  - `link` - a link that triggers an action

##### Properties

- `background`
- `border`
- `text`
- `focus-ring`
- `padding-inline`
- `padding-block`
- `focus-ring`
